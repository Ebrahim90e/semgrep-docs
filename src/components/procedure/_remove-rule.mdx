To **remove a rule** from the Rule board:

1. In Semgrep Cloud Platform, click **[Rule board](https://semgrep.dev/orgs/-/board)**.
2. Click the ruleset that contains the rule.
3. Click the <i className="fa-solid fa-trash-can inline_svg"></i> **Remove ruleset** icon next to the rule you're deleting.
4. Click **Save**.

:::info
- Individual rules within rulesets can only be disabled, not deleted. To disable an individual rule in **[Rule board](https://semgrep.dev/orgs/-/board)**, click the <i class="fa-solid fa-toggle-large-on"></i> toggle to disable the rule. See also [Disabling rules](/semgrep-app/rule-board/#disabling-rules).
- When you remove a rule from the Rule Board, all associated findings on Findings page and Dashboard page are removed also.
:::

Remove a rule on the Code page of Semgrep Cloud Platform. To do so, follow these steps:

1. Go to the Semgrep Cloud Platform [Code](https://semgrep.dev/orgs/-/findings?tab=open) page.
1. Select the checkbox on the left side of a finding that was detected by a rule you want to ignore.
1. Click **Triage**, and then under **Triage state** select **Ignore**.
1. Optional: Select a reason of why you are ignoring a finding. Choose either: **False positive**, **Acceptable risk**, **No time to fix**
1. Click **Ignore**.
1. Optional: Select whether you want to ignore all findings in **Just this file**, **This directory**, or **Parent directory**.
1. Enable the checkbox to: **Remove this rule from Rule board**. This removes the related rule that matched the finding.
