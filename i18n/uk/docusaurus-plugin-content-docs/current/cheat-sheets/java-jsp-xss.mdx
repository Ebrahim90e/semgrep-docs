# Запобігання XSS (міжсайтового скриптингу) для Java і JSP (Сторінки сервісу Джава)
​
​
Це шпаргалка (підказка) для запобігання міжсайтовим скриптам (XSS) від r2c. Він містить закономірності (словосполучення/норми/схеми) коду потенційного XSS у програмі. Замість того, щоб ретельно перевіряти код на наявність недоліків, якими можуть скористатися, рекомендації в цій шпаргалці пропонують безпечну дорогу для розробників, яка зменшує можливість XSS у вашому коді. Дотримуючись цих рекомендацій, ви можете бути достатньо впевнені, що ваш код не містить XSS.
​
## Підсумок запобіжних дій
​
JSP під загрозою. Ігнорування HTML у шаблонах JSP вимагає ігноруваня всіх даних, які відображаються на сторінці. Що ще гірше, бізнес-логіку можна вбудувати в JSP за допомогою скриптлетів. Це легко забути і цим легко зловживати, і це може легко зробити XSS уразливими. Опція за замовчуванням має бути безпечним варіантом: подумайте про використання процесори вебшаблонів , які ігнорують HTML за замовчуванням, наприклад, JSF або Velocity. Якщо ви не можете перейти на інший фреймворк (модель), використовуйте спеціальний(індивідуально зроблений) арбітр EL (скриптова мова виразів), який ігнорує HTML за замовчуванням в JSP, наприклад https://docs.oracle.com/javaee/6/api/javax/el/ELResolver.html, інакше ви ПОВИННІ переконайтеся, що всі дані ігноруються. Не використовуйте скриптлети.
